## 2. Цель проекта

Создать простого бота для MAX, который отправляет приветственное сообщение с полезными кнопками. Бот должен работать без ИИ, без хранения данных, без многошаговых сценариев и без продаж внутри чата. Основной фокус на стабильности и простоте.

## 3. Функциональные требования

### 3.1. Основной сценарий работы

1. Пользователь открывает бота в MAX.
2. Пользователь нажимает кнопку **Старт** / **Начать** (или отправляет `/start`).
3. Бот отправляет **одно приветственное сообщение**.
4. Под сообщением отображаются **3 inline-кнопки**:
   - **Виртуальный шоурум**
   - **Контакты: +7 (950) 730-72-02**
   - **Адрес шоурума**

### 3.2. Поведение кнопок

#### Кнопка 1: "Виртуальный шоурум"
- При нажатии открывается ссылка: `https://room-set.ru/virtualnyj-shourum/`
- Тип: inline_keyboard с link

#### Кнопка 2: "Контакты: +7 (950) 730-72-02"
- Приоритет: При нажатии открывается телефонная ссылка: `tel:+79507307202`
- Запасной вариант: Если MAX не поддерживает `tel:` в кнопках, использовать ссылку на страницу контактов: `https://room-set.ru/kontakty/`
- В тексте сообщения номер телефона должен быть написан явно: **+7 (950) 730-72-02**

#### Кнопка 3: "Адрес шоурума"
- При нажатии открывается ссылка на карту: `https://maps.google.com/?cid=3523281962486703769&g_mp=CiVnb29nbGUubWFwcy5wbGFjZXMudjEuUGxhY2VzLkdldFBsYWNlEAIYASAA&hl=ru-RU&source=embed`
- Тип: inline_keyboard с link

### 3.3. Текст приветственного сообщения

Бот должен отправлять следующий текст:

```
Здравствуйте! 👋

Добро пожаловать в ROOMSET.

Здесь вы можете быстро:
• открыть виртуальный шоурум;
• связаться с нами по телефону;

Телефон: +7 (950) 730-72-02
Адрес шоурума: г. Челябинск, ул. Чайковского, 173
```

## 4. Технические требования

### 4.1. Архитектура
- Язык: Python 3.11+
- Фреймворк: FastAPI + uvicorn
- Деплой: Railway
- API: MAX API (webhook)

### 4.2. Обработка событий
- Обработка команды `/start` или события `bot_started`
- Обработка события `message_created` с текстом `/start` или `/старт`

### 4.3. Отправка сообщений
- Использовать MAX API для отправки сообщений
- Сообщение должно содержать текст + inline-кнопки
- Кнопки: тип `inline_keyboard` с `payload.buttons`

### 4.4. Требования к коду
- Максимальная простота и читаемость
- Все параметры (текст, ссылки, номер телефона) должны быть легко изменяемыми константами в начале файла
- Нет хранения данных пользователя
- Нет сложной логики (только один сценарий)
- Код должен быть документирован комментариями

### 4.5. Тестирование
- Локальное тестирование через curl/Invoke-RestMethod
- Тестирование на Railway (логи, публичный URL)
- Тестирование в MAX на реальном устройстве (проверка открытия ссылок)

### 4.6. Безопасность
- Токен MAX не хранится в коде (только в переменных окружения)
- HTTPS обязателен для webhook
